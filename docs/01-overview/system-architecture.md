# Foggy Navigator 整体架构设计

## 1. 系统概述

### 1.1 系统定位

Foggy Navigator 是一个基于 LangChain4j 的企业级动态 Agent 编排系统，旨在提供强大的 AI 对话和任务执行能力。系统支持多 Agent 管理、智能任务编排、持久化记忆、知识库检索、工具调用和人工确认等核心功能。

### 1.2 核心价值

- **动态 Agent 编排**：支持用户自定义和系统预制的 Agent，灵活配置
- **智能任务执行**：自动分解复杂任务，支持动态调整和中断恢复
- **持久化记忆**：短期和长期记忆结合，支持记忆压缩和关联
- **知识库检索**：RAG 技术增强，支持多种检索策略
- **工具调用**：内置工具和 MCP 工具支持，支持 HITL 人工确认
- **可观察性**：实时监控会话和任务状态，支持任务控制
- **企业级特性**：多租户、权限控制、审计日志、告警机制

### 1.3 技术栈

- **核心框架**：LangChain4j
- **编程语言**：Java 17+
- **数据库**：MySQL 8.0+（关系型数据）、Redis（缓存）
- **向量数据库**：Milvus / Qdrant / Weaviate（可选）
- **消息队列**：RabbitMQ / Kafka（异步任务）
- **监控**：Prometheus + Grafana
- **日志**：ELK Stack（Elasticsearch + Logstash + Kibana）

```
    └── File Storage（MinIO）
    ↓
响应返回
```

## 3. 核心模块

### 3.1 Agent 管理模块

**职责**：管理 Agent 的创建、配置、存储和选择

**核心功能**：
- Agent 定义和配置
- Agent 能力描述
- Agent 选择策略
- Agent 权限管理

**数据模型**：
- Agent（Agent 定义）
- AgentCapability（Agent 能力）
- AgentConfig（Agent 配置）

**API**：
- 创建/更新/删除 Agent
- 查询 Agent 列表
- 选择合适的 Agent

### 3.2 Session 管理模块

**职责**：管理对话会话的生命周期和上下文

**核心功能**：
- Session 创建和恢复
- 消息存储和检索
- 上下文管理
- 上下文压缩

**数据模型**：
- Session（会话）
- Message（消息）
- SessionContext（会话上下文）

**API**：
- 创建/关闭 Session
- 发送/接收消息
- 查询 Session 历史

### 3.3 Task 编排模块

**职责**：管理任务的分解、执行和动态调整

**核心功能**：
- 任务分解和规划
- 任务执行监控
- 动态任务调整
- 任务中断和恢复

**数据模型**：
- TaskPlan（任务计划）
- TaskStep（任务步骤）
- TaskExecution（任务执行）

**API**：
- 创建任务计划
- 执行任务
- 暂停/继续/中止任务

### 3.4 Memory 模块

**职责**：管理短期和长期记忆的存储和检索

**核心功能**：
- 短期记忆读写
- 长期记忆检索
- 记忆压缩和优化
- 记忆关联管理

**数据模型**：
- MemoryEntry（记忆条目）
- MemoryType（记忆类型）
- MemoryAssociation（记忆关联）

**API**：
- 添加/查询记忆
- 检索相关记忆
- 压缩记忆

### 3.5 RAG 模块

**职责**：管理知识库检索和上下文注入

**核心功能**：
- 知识库检索
- 上下文注入
- 检索策略优化
- 检索结果处理

**数据模型**：
- KnowledgeBase（知识库）
- Document（文档）
- RAGResult（检索结果）

**API**：
- 检索知识
- 注入上下文
- 管理知识库

### 3.6 Tool 模块

**职责**：管理工具的注册、调用和隔离

**核心功能**：
- 工具注册和发现
- 工具调用执行
- HITL 人工确认
- 工具隔离管理

**数据模型**：
- ToolDefinition（工具定义）
- ToolExecution（工具执行）
- HITLRequest（人工确认请求）

**API**：
- 注册/注销工具
- 调用工具
- 管理 HITL

### 3.7 可观察性模块

**职责**：监控系统状态、收集指标、提供控制接口

**核心功能**：
- 会话监控
- 任务监控
- 指标收集
- 告警管理
- 日志追踪

**数据模型**：
- SessionSnapshot（会话快照）
- TaskSnapshot（任务快照）
- Metrics（指标）
- Alert（告警）

**API**：
- 查询会话/任务状态
- 控制任务（暂停/继续/中止）
- 查询指标和告警

## 4. 技术选型

### 4.1 核心框架

**LangChain4j**
- 优势：Java 生态、与 Spring Boot 集成良好、支持多种模型
- 用途：Agent 框架、工具调用、模型适配

### 4.2 数据库

**MySQL 8.0+**
- 优势：成熟稳定、ACID 事务、丰富的索引
- 用途：存储结构化数据（Agent、Session、Task、Tool 等）

**Redis**
- 优势：高性能、支持多种数据结构
- 用途：缓存、会话状态、分布式锁

**Milvus / Qdrant / Weaviate**
- 优势：高性能向量检索、支持大规模数据
- 用途：长期记忆存储、知识库检索

### 4.3 消息队列

**RabbitMQ / Kafka**
- 优势：异步处理、解耦系统、削峰填谷
- 用途：异步任务、事件驱动

### 4.4 监控和日志

**Prometheus + Grafana**
- 优势：强大的指标收集和可视化
- 用途：系统监控、性能分析

**ELK Stack**
- 优势：完整的日志解决方案
- 用途：日志收集、查询、分析

## 6. 安全架构

### 6.1 认证和授权

- **认证**：JWT Token、OAuth 2.0
- **授权**：RBAC（基于角色的访问控制）
- **API 安全**：API Key、Rate Limiting

### 6.2 数据安全

- **传输加密**：HTTPS/TLS
- **存储加密**：敏感数据加密存储
- **数据脱敏**：日志和监控中的敏感信息脱敏

### 6.3 审计和合规

- **审计日志**：记录所有关键操作
- **数据保留**：根据合规要求设置保留策略
- **隐私保护**：遵循 GDPR、CCPA 等法规

## 7. 性能和可扩展性

### 7.1 性能优化

- **缓存策略**：多级缓存（本地缓存 + Redis）
- **数据库优化**：索引优化、查询优化、读写分离
- **异步处理**：使用消息队列处理耗时任务

## 10. 总结

Foggy Navigator 是一个功能完善的企业级动态 Agent 编排系统，采用分层架构设计，包含表现层、API 层、编排层、业务模块层、可观察性层和基础设施层。系统基于 LangChain4j 框架，使用 MySQL、Redis、Milvus 等成熟技术栈，支持分布式部署和高可用架构。

通过合理的模块划分和清晰的依赖关系，系统具有良好的可扩展性和可维护性。安全架构、性能优化、灾备和高可用等设计确保了系统的稳定性和可靠性。

本文档为系统的整体架构提供了完整的视图，为后续的详细设计和实施提供了基础。
