# PC Workers 操作与体验检查清单

## 1. 目标

这份清单用于桌面端 `Workers` 页面日常操作、自测回归和 Playwright 体验验证。

覆盖对象：

- `packages/navigator-frontend/src/views/ClaudeWorkerView.vue`
- `packages/navigator-frontend/src/composables/useClaudeWorker.ts`
- `packages/navigator-frontend/src/composables/useForwardSession.ts`
- `packages/navigator-frontend/src/composables/useAttachments.ts`
- `packages/navigator-frontend/src/api/unifiedTask.ts`

重点覆盖近期易变区域：

- 目录树与工作区切换
- 任务创建与继续对话
- 消息转发到新会话/已有会话
- 图片/文件附件
- 里程碑、会话同步、Git 同步
- 文件浏览器、终端、Worktree

## 3. 页面操作手册

### 3.1 进入页面

1. 登录系统。
2. 打开 `/#/`。
3. 确认左侧展示 Worker 树。

### 3.2 Worker 与目录树

1. 展开一个 `ONLINE` Worker。
2. 确认目录树按下面顺序显示：
   - 项目目录
   - 项目子目录
   - worktree
   - 孤儿目录
3. 点击目录后，确认中间区域显示：
   - 目录名
   - Git 分支
   - Git 状态
   - 实际路径
   - 生效中的 Auth/模型配置标识（如 `API: test`）

### 3.3 目录头部动作

选中目录后，逐项确认入口存在并可点击：

- `同步 Git`
- `同步会话`
- `创建 Worktree`（非项目目录且非 worktree 时）
- `浏览文件`
- `VS Code`
- `终端`
- `全屏`
- `里程碑`
- `编辑`

### 3.4 创建任务

1. 在输入框输入普通 prompt。
2. 可选设置：
   - 模型
   - 权限模式
   - 最大轮次
   - Agent Team
3. 点击发送。

预期：

- 新任务出现在当前目录会话区
- 右侧历史列表出现新会话
- 任务完成后可继续发送下一轮

### 3.5 附件输入

主输入框应支持：

- 粘贴截图
- 拖拽图片
- 拖拽普通文件
- 文件选择器添加文件

预期：

- 图片显示缩略图
- 普通文件显示文件卡片
- 可单个移除
- 发送后附件随消息提交

### 3.6 转发会话

从已完成会话中选择一条可转发消息，验证两条链路：

#### 转发到新会话

1. 打开“转发为新会话”弹窗。
2. 检查默认带出：
   - 源消息内容
   - Worker
   - 目录
   - cwd
   - 模型/模型配置
   - 权限模式
   - 里程碑（同目录时）
3. 在转发输入框内追加文本。
4. 再测试一次粘贴截图或拖拽文件。
5. 提交。

预期：

- 新会话创建成功
- 新会话自动打开
- 附件可随转发一起提交

#### 转发到已有会话

1. 选择“转发到已有会话”。
2. 检查下拉中只出现允许追加的目标会话。
3. 确认目标摘要显示：
   - 会话
   - Worker
   - 目录
   - Provider
   - 模型配置
   - 模型
   - 里程碑
4. 提交。

预期：

- 内容成功追加到目标会话
- 当前页面可切到该会话继续查看

### 3.7 会话历史操作

至少验证以下能力：

- `继续`
- `重连`
- `重新同步`
- `回退到此`
- `设置里程碑`
- `编辑标题`
- `批量绑定 Auth`（如环境允许）

重点预期：

- 回退后输入框能回填原 prompt
- 继续发送后会话可正常完成
- 里程碑更新后历史分组同步变化

### 3.8 文件浏览器

1. 点击 `浏览文件`。
2. 确认新标签页能打开对应目录。
3. 尝试打开一个文档文件。

预期：

- 根目录树可加载
- 文件内容可展示
- deeplink 打开场景可定位到目标文件

### 3.9 终端

1. 点击 `终端`。
2. 如 Worker 已配 SSH，验证能直接连；否则手动输入 SSH 信息。

预期：

- 终端面板出现
- 可创建 tab
- 可关闭 tab
- 面板最小化/恢复正常

### 3.10 Worktree

1. 在非 worktree 目录点击 `创建 Worktree`。
2. 输入新分支名并创建。

预期：

- 左侧树新增 worktree 节点
- 新节点显示 `worktree` 标签
- Git 分支信息正确

## 4. Playwright 执行清单

执行浏览器自动化时，建议按下面顺序：

| 序号 | 场景 | 必测 |
|------|------|------|
| 1 | 登录并进入 `/#/` 的 `Workers` 页面 | 是 |
| 2 | 展开 Worker，选择目录 | 是 |
| 3 | 验证目录头部动作区 | 是 |
| 4 | 验证目录路径、Git 分支、Git 状态、生效 Auth 展示 | 是 |
| 5 | 创建一条简单任务并等待完成 | 是 |
| 6 | 从历史消息打开转发弹窗 | 是 |
| 7 | 转发到新会话 | 是 |
| 8 | 转发到已有会话 | 是 |
| 9 | 转发弹窗内附件预览与移除 | 是 |
| 10 | 打开文件浏览器 | 是 |
| 11 | 打开终端入口 | 建议 |
| 12 | 同步 Git / 同步会话按钮可用性 | 建议 |
| 13 | 里程碑入口与弹窗 | 建议 |
| 14 | Worktree 创建入口显隐 | 建议 |
| 15 | 新建会话后右侧历史会话立即出现新卡片 | 是 |
| 16 | 会话运行中时，中间主会话区与右侧历史卡片状态一致 | 是 |
| 17 | 会话完成后，中间主会话区与右侧历史卡片状态同步切换 | 是 |

### 4.1 本轮新增 BUG 回归点

本轮 Playwright 自动验证必须额外覆盖以下回归点：

1. 新建会话成功后，右侧“历史会话”无需等待任务完成即可出现新卡片。
2. 会话创建后的 `处理中` 状态，在中间主会话区与右侧历史卡片之间保持一致。
3. 会话完成后，右侧历史卡片不应残留 `处理中` 状态。

关联 BUG：

- [18-workers-history-session-refresh-and-status-sync-bug.md](/D:/foggy-projects/Foggy-Navigator-wt-qd-win11-dev/docs/version-tracker/1.0.1-SNAPSHOT/18-workers-history-session-refresh-and-status-sync-bug.md)

## 6. 已知注意点

- `Workers` 页面依赖真实 Worker、目录和会话数据，空环境下很多入口无法形成闭环。
- 转发到已有会话只适用于允许追加的目标会话，不是任意历史会话都可选。
- 附件能力依赖当前前后端都支持 `images` 字段透传。
- 终端、Worktree、文件浏览器都对目录配置正确性敏感，路径异常会直接导致假失败。
