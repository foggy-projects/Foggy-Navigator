# 编程 Agent 集成

> 当前有效范围：`addons/coding-agent` 独立模块的能力说明。
>
> 本文档不再把“语义层 JavaScript 自动编辑”视为当前主产品主线，而是把 `coding-agent` 说明为一个可独立部署、也可被主平台引用的编程执行模块。

## 1. 模块定位

`addons/coding-agent` 的职责是管理编程会话及其执行环境，面向 OpenHands 一类执行引擎提供统一的接入外壳。它更接近“编程执行子系统”，而不是当前主平台的唯一交互入口。

在当前仓库中，它主要承担以下职责：

- 管理编程会话、消息、事件流与恢复流程
- 管理执行容器和环境生命周期
- 管理 Git 凭证、仓库访问与基础验证流程
- 为独立前端或主平台嵌入提供后端 API

它与主平台的关系是：

- 可以独立运行，形成自己的会话与环境管理闭环
- 可以被主前端引用，作为某些编程工作流的后端能力来源
- 不负责统一产品导航、跨模块编排总入口和整个平台治理口径

## 2. 当前已实现的能力面

从 `addons/coding-agent/src/main/java` 当前接口可以确认，这个模块已经形成了比较完整的 API 面：

### 2.1 会话与消息

- `ConversationController`
- `MessageController`
- `EventController`
- `ConversationRecoveryController`

对应能力包括：

- 创建、查询、删除编程会话
- 向会话发送消息并拉取消息历史
- 通过 SSE 或轮询读取会话事件
- 对异常中断的会话做恢复或清理

### 2.2 执行环境与容器

- `EnvironmentController`
- `ContainerController`

对应能力包括：

- 创建和销毁执行环境
- 查询环境状态
- 查询和清理容器
- 把“会话”和“底层执行环境”做显式解耦

### 2.3 Git 与凭证

- `GitCredentialController`
- `GitProviderController`

对应能力包括：

- 管理 Git 凭证
- 测试凭证可用性
- 读取项目、分支等仓库信息
- 为编程会话准备代码仓上下文

### 2.4 验证与全局事件

- `ValidationController`
- `GlobalEventController`

对应能力包括：

- 对指定会话发起验证
- 查询验证结果
- 获取全局事件列表与 SSE 流

## 3. 当前架构理解

当前更准确的理解方式不是“CodingAgent 控制一个专门的语义层修复机器人”，而是：

1. 前端或上层平台创建编程会话
2. 会话绑定执行环境、容器和 Git 上下文
3. 用户持续发送消息或指令
4. 底层执行引擎产生日志、消息、事件和状态变化
5. 模块通过 API 与 SSE 将状态暴露给上层界面

因此，这个模块的核心价值在于“执行上下文管理”和“编程会话生命周期管理”，不是某一个特定领域模型编辑器。

## 4. 与主前端能力面的关系

放在当前产品口径下，这个模块通常作为以下能力的后端来源：

- Worker/项目 Agent 的编程执行上下文
- 会话中的编程任务落地执行
- 跨项目任务中的具体代码操作节点
- 独立编程工作台或嵌入式编程面板

它不直接代表以下内容：

- 当前主产品整体 IA 与功能地图
- 平台级任务治理总线
- Session 主模块的通用会话抽象
- 所有 Agent 形态的统一实现

## 5. 当前边界与非目标

结合现有代码和主文档口径，当前不应再把以下内容写成既成事实：

- 所有编程任务都必须经由 `coding-agent`
- 当前产品的核心场景仍然是 TM/QM 或语义层 JS 编辑
- 验证服务一定是语义层专用验证器
- OpenHands 是唯一执行引擎且已经完全锁定
- 自动提交 PR 是所有流程的默认终态

更合理的表述是：

- `coding-agent` 当前偏向 OpenHands 风格执行环境集成
- 其验证、环境、Git、事件流机制具有复用价值
- 具体业务域校验器、提交策略和工作流终态可以因场景而变化

## 6. 设计遗产与仍然有效的部分

旧版本文档中，以下内容仍有参考价值：

- 独立环境/容器执行的隔离思路
- 会话与工作目录绑定的思路
- 验证反馈进入会话闭环的思路
- Git 凭证与仓库上下文前置准备的思路

以下内容则主要属于历史设计，不再代表当前主产品方向：

- 以“语义层文件自动编辑”为唯一目标
- 围绕 TM/QM 技能注入展开的产品叙事
- 把主系统整体等同于 OpenHands 集成壳层
