# 工作区与 Worker 中心

## 1. 功能定位

这是当前系统最重的操作台，负责把远程编程资源和任务执行资源统一暴露给用户。

对应前端入口：

- `Workers`
- `File Browser`

对应后端模块：

- `addons/claude-worker-agent`
- `addons/codex-worker-agent`
- `session-module`

## 2. 功能范围

### 2.1 Worker 管理

- 注册、编辑、删除 Worker
- Worker 健康检查
- 读取 Worker 进程
- 终止指定进程
- 获取 code-server 访问密码
- 同步 Worker Skills

### 2.2 工作目录管理

- 创建标准目录与项目目录
- 编辑、删除目录
- 查看目录树关系
- 创建和删除 Git worktree
- 同步目录 Git 信息
- 查询目录下可用 Skills

### 2.3 目录内任务操作

- 在目录上下文中发起任务
- 为任务指定权限模式、模型、团队配置
- 回复、重连、回溯、恢复任务
- 同步目录历史会话
- 查看目录级任务分页与搜索结果

### 2.4 文件与 Git 浏览

- 文件树浏览
- 文件内容预览
- 文件名搜索与内容搜索
- Git diff 摘要
- 单文件 diff
- Git 历史提交列表
- 提交详情与单文件提交 diff
- 忽略规则管理

## 3. 设计特点

### 3.1 工作区是任务的天然上下文

任务不是悬空执行的，而是绑定到：

- Worker
- 目录
- 可能的 worktree
- 可能的 Agent Team
- 可能的模型配置

### 3.2 界面结构是树形资源组织

前端已经形成四层资源关系：

1. Worker
2. 项目目录
3. 子目录
4. worktree

### 3.3 文件浏览器不是编辑器，而是审阅与定位工具

当前主能力是：

- 浏览
- 搜索
- 对比
- 历史审查

不是本地 IDE 替代品。

## 4. 典型使用场景

### 场景 1：接入一台新的远程 Worker

1. 在设置页注册 Worker
2. 在 Workers 页展开该 Worker
3. 新建项目目录
4. 同步目录状态并开始创建任务

### 场景 2：在指定目录执行编程任务

1. 选择目录
2. 输入 Prompt
3. 选择权限模式和模型
4. 创建任务
5. 在历史区持续跟踪与交互

### 场景 3：审查代码变更

1. 打开文件浏览器
2. 切换到 Git 改动
3. 查看 diff
4. 切换到 Git 历史查看提交与文件差异

## 5. 与其他功能域的关系

- 为 `任务治理中心` 提供底层任务来源
- 为 `跨项目编排` 提供目录、分支、阶段执行上下文
- 为 `平台设置与资源治理` 提供 Worker 管理对象
