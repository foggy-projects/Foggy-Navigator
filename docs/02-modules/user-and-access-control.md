# 用户与访问控制

## 1. 功能定位

该能力负责平台身份认证、用户治理和对外 API Key 管理。

对应前端入口：

- `登录`
- `用户`

对应后端模块：

- `user-auth-module`

## 2. 功能范围

### 2.1 登录认证

- 用户注册
- 用户登录
- 当前用户信息读取
- JWT 鉴权

### 2.2 用户管理

- 新增用户
- 编辑用户基础资料
- 修改角色
- 修改状态
- 删除用户

### 2.3 API Key 管理

- 为用户创建 API Key
- 查看脱敏 Key 列表
- 设置过期时间
- 撤销 Key
- 查看最近使用时间

## 3. 当前角色模型

前端已体现的角色包括：

- `SUPER_ADMIN`
- `TENANT_ADMIN`
- `DEVELOPER`
- `VIEWER`

这说明平台具备基础的多角色治理结构，即使当前主要使用方式仍偏单租户/内部平台。

## 4. 设计特点

### 4.1 登录是全平台统一前置

除公开能力外，前端主路由默认都受登录状态约束。

### 4.2 用户管理更偏平台治理，不是社交型用户中心

当前重点是：

- 账户可用性
- 角色权限
- 对外调用凭证

### 4.3 API Key 是对外集成入口的一部分

用户 API Key 与 Open API、SDK 等对外能力天然关联。

## 5. 典型使用场景

- 管理员新增内部用户
- 管理员调整用户角色和状态
- 为脚本或外部系统生成 API Key
- 撤销失效或泄露的 API Key
