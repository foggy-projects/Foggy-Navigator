# 任务治理中心

## 1. 功能定位

任务治理中心负责从平台视角统一管理任务，而不是只在单个会话里看消息。

对应前端入口：

- `任务`

对应后端模块：

- `session-module`
- `addons/claude-worker-agent`
- `addons/codex-worker-agent`

## 2. 功能范围

### 2.1 统一任务分发

入口：

- `POST /api/v1/tasks`

作用：

- 根据 providerType、modelConfigId、agentId 选择执行后端
- 兼容 Direct Route 与 A2A Route
- 统一输出任务视图

### 2.2 任务查询与治理

- 查询任务详情
- 分页查询
- 搜索任务
- 按目录查询
- 按 Worker 查询
- 查询消息数量与消息内容

### 2.3 任务控制

- 取消
- 恢复
- 重连
- 重同步
- 回溯
- 扫描 checkpoint

## 3. 当前任务类型理解

当前前端看板已经识别出至少三类任务：

- `CODING`
- `CLAUDE_WORKER`
- `DELEGATION`

这说明任务层已经不仅是单一编程任务，而是平台级任务域。

## 4. 设计特点

### 4.1 任务中心和会话中心是两种视角

- 会话中心强调“对话连续性”
- 任务中心强调“执行对象、状态和治理动作”

### 4.2 统一分发比 addon 直调更重要

当前架构已经把任务创建收束到 `TaskDispatchFacade`，这是平台一致性的关键：

- 路由规则统一
- 会话绑定统一
- DTO 统一
- 任务控制统一

### 4.3 任务中心承担平台治理能力

它不只是展示列表，还承担：

- 执行恢复
- 失败补救
- 对账同步
- 过程审计

## 5. 典型使用场景

### 场景 1：查看所有正在运行的任务

用户进入任务看板后，可以按状态和目标 Agent 快速过滤任务。

### 场景 2：处理中断任务

当任务连接断开或状态异常时，用户可以执行重连、恢复或重同步，而不是重新发起任务。

### 场景 3：从任务追溯到执行上下文

用户可以进一步查看任务所属目录、Worker 会话、消息记录和 checkpoint。
